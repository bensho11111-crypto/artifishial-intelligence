# Synthetic Lowrance Elite 7 Data

Two minutes of a fishing boat trolling across Puddingstone Reservoir, CA
(33.90°N, 117.50°W). Generated by `generate.py`.

## Files

| File | Format | Size | Description |
|------|--------|------|-------------|
| `nmea_0183.nmea` | ASCII text | 67 KB | NMEA 0183 serial stream |
| `track.geojson` | GeoJSON | 50 KB | GPS track with per-point depth/heading |
| `sample.sl2` | Binary | 66 KB | StructureScan SL2 (120 packets, primary 200 kHz) |

## How to use with the app

```bash
# Replay SL2 file
python src/main.py --sl2 data/synthetic/sample.sl2

# The NMEA file can be fed via a virtual serial port (e.g. com0com on Windows)
# or used as reference for building a file-based NMEA reader
```

## Data details

### nmea_0183.nmea

Interleaved NMEA 0183 sentences exactly as the Lowrance Elite 7 outputs
them on its USB serial port (38400 baud, 8N1). Sentence mix per second:

```
$SDDBT  — depth below transducer (feet / metres / fathoms)     5 Hz
$SDDPT  — depth + transducer offset                             5 Hz
$HEHDT  — true heading (degrees)                                5 Hz
$GPGGA  — GPS fix (lat/lon/HDOP/altitude)                       1 Hz
$GPRMC  — GPS recommended minimum (speed/course/date)           1 Hz
$GPVTG  — course and speed over ground                          1 Hz
$SDMTW  — water temperature                                    ~0.1 Hz
```

All checksums are valid (XOR of bytes between `$` and `*`).

### track.geojson

A GeoJSON FeatureCollection with:
- One `LineString` of the full GPS track
- One `Point` per GPS fix (120 total) with properties:
  `t_s`, `depth_m`, `heading_deg`, `speed_kts`, `hdop`

Depth range: 1.6–11.4 m (mean 6.0 m) — realistic for this reservoir.

### sample.sl2

Binary SL2 format. Each 552-byte packet contains:
- 40-byte header (depth, temp, speed, signal strength, channel ID, etc.)
- 512-byte sonar echo array (A-scope amplitude, 0–255 per range cell)

The echo data includes a realistic Gaussian bottom return, a second-bounce
multi-path echo at 2× depth, and occasional weak fish-arch returns.

## How representative is this?

### What matches real hardware

- **Sentence types** — the exact sentences the Elite 7 outputs at default settings
- **GPS rate** — 1 Hz is the Lowrance hardware default
- **Sonar rate** — 5 Hz is within the real 5–15 Hz range (varies with depth range setting)
- **Depth variability** — 3-component sinusoidal model gives plausible lake-floor profiles
- **Speed range** — 2.8–4.3 kts matches typical trolling
- **Signal strength** — 140–225 counts, dropping slightly at higher speed, matching real patterns
- **SL2 block layout** — offsets for depth (24), temperature (28), and signal strength (36)
  match what `StructureScanReader._decode_block` expects and have been validated by replay
- **Multi-path echo** — second bottom return at 2× depth is a real sonar artefact
- **Fish arches** — appear in ~25 % of pings, as they do on a typical fishing lake

### What differs from real hardware

1. **No GPS position noise** — real receivers have 2–5 m CEP jitter that appears
   as a random walk in the fix stream. These positions move perfectly smoothly.

2. **No NMEA corruption** — real serial streams occasionally have checksum failures,
   partial lines (mid-sentence buffer overflows), and non-ASCII bytes. A production
   parser needs to handle these gracefully.

3. **No HDOP spikes** — in a real canyon or under tree cover, HDOP can jump to
   3–8 and GPS fixes drop to 6 satellites. The synthetic HDOP stays near 0.8–1.2.

4. **Simplified sonar echo** — real SL2 echo arrays contain the full broadband
   A-scope return including vegetation clutter, suspended sediment, bubbles from
   the transom, and interference between the 83 kHz and 200 kHz beams. The
   synthetic version is a clean Gaussian + noise.

5. **No sidescan channels** — a real SL2 from an Elite 7 with StructureScan
   contains channel IDs 3 (port) and 4 (starboard) sidescan in addition to
   channel 0 (primary downward). This file only has channel 0.

6. **Speed-dependent sonar degradation** — above ~7 knots, cavitation under the
   hull causes loss-of-bottom. Not modelled here.

7. **No temperature sensor drift** — real thermistors show slow drift and lag
   (sensor time constant ~30 s). The synthetic temperature is nearly constant.

### To get real data

- **NMEA serial**: Connect the Elite 7's USB port. It appears as a COM port
  (Windows) or `/dev/ttyUSB0` (Linux). Capture at 38400 baud, 8N1 with PuTTY
  or `python -m serial.tools.miniterm COMx 38400`.

- **SL2 files**: Insert a microSD card. Navigate the unit to
  *Files → This Device → [Session Name]* and the `.sl2` file will be on the card.
  Files are typically 5–200 MB per session.
